📚 FRP 客户端全景配置手册

整合：纯客户端基础配置、Store 持久化、宝塔面板多域名反代全解。

一、 基础配置（全局区）

基础配置必须放在配置文件的最顶部，不可被包含在任何 [[proxies]] 内部。

serverAddr = "1.2.3.4"
serverPort = 7000

auth.method = "token"
auth.token = "strong_token_here"

transport.heartbeatInterval = 30
transport.heartbeatTimeout = 90

二、 Store 持久化配置 (v0.52+ 核心)

🧠 一句话总结：store 是新版 FRP 的“记忆芯片”，配置它，你的客户端重启后不再“失忆”。用于保存 OIDC Token、访客端身份信息等。

1️⃣ 使用本地文件存储（推荐 & 最简单）

如果不配置 store，重启 frpc 后可能需要重新认证。

# 客户端本地存储配置
[store]
type = "local"
# 本地文件路径（请确保 frpc 有权限读写此目录）
localPath = "/data/adb/modules/CHERWIN_FRPC/conf/frpc_data"
  • 路径可以是相对路径或绝对路径，文件夹不存在时会自动创建。
  • 迁移设备时，拷贝此目录下的文件可保留会话状态。

2️⃣ 使用 Redis 存储（高可用/多实例场景）

[store]
type = "redis"
redisAddr = "127.0.0.1:6379"
redisDB = 0
redisPassword = "redis_password" # 无密码则注释
redisPrefix = "frpc_client_01"   # Key前缀，区分不同客户端

四、 宝塔面板 + 多域名 Web 反代实战

🎯 整体架构：浏览器 ➔ 80/443 (Nginx/Apache) ➔ frps (9080) ➔ frpc ➔ 内网服务
✅ 优点：不用修改系统保留端口、支持 SSL 证书、支持 WebSocket、支持无限域名绑定。

1️⃣ 服务端 frps 配置（宝塔端）

📌 注意：不要使用 80 / 443，这些端口应留给 Web 服务自身。
bindPort = 7000
vhostHTTPPort = 9080
vhostHTTPSPort = 9443
# ...鉴权配置略...

2️⃣ Nginx 反向代理配置（必须）

在宝塔创建站点（如 app.example.com），设置纯静态，进入 反向代理 -> 配置文件：

📌 核心：缺少 Host 头会导致 frps 返回 404 powered by frp。
location / {
    proxy_pass http://127.0.0.1:9080;
    
    # 核心：传递真实域名，FRP 只认这个
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # ✅ WebSocket 支持
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

3️⃣ Apache 反向代理配置

📌 核心：不写 ProxyPreserveHost On 必现 404。
#PROXY-START/
<IfModule mod_proxy.c>
    ProxyRequests Off
    SSLProxyEngine on

    # ✅ 核心：传递真实域名
    ProxyPreserveHost On

    ProxyPass / http://127.0.0.1:9080/
    ProxyPassReverse / http://127.0.0.1:9080/

    # ✅ WebSocket 支持
    RewriteEngine On
    RewriteCond %{HTTP:Upgrade} websocket [NC]
    RewriteCond %{HTTP:Connection} upgrade [NC]
    RewriteRule ^/?(.*)$ ws://127.0.0.1:9080/$1 [P,L]
</IfModule>
#PROXY-END/

4️⃣ 常见排错清单

错误现象根本原因解决方案
404 powered by frpHost 头丢失检查 Nginx/Apache 中是否透传了 Host ($host)
WebSocket 连不上未开启 Upgrade在反代配置文件中添加协议升级头配置
HTTPS 无法访问证书未配置直接在宝塔站点中申请 Let's Encrypt，然后正常反代到 9080
只能 HTTP 不能 HTTPS端口未映射检查服务端配置是否包含 vhostHTTPSPort
FRP 不认域名，只认 HTTP Host 头。
谁挡住了 Host，谁就是 404 的凶手。
