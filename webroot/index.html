1. 基础配置

连接 FRP 服务端需配置以下参数：

serverAddr = "your-server.com"  # 服务器公网 IP 或域名
serverPort = 7000               # 与服务端 bindPort 一致
auth.method = "token"
auth.token = "your_strong_token"

2. 代理通道类型

TCP — SSH、RDP、数据库等：

[[proxies]]
name = "ssh"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 6000

UDP — DNS 解析、游戏等：

[[proxies]]
name = "dns"
type = "udp"
localIP = "127.0.0.1"
localPort = 53
remotePort = 6001

HTTP — Web 服务：

[[proxies]]
name = "web"
type = "http"
localIP = "127.0.0.1"
localPort = 8080
customDomains = ["www.example.com"]

HTTPS：

[[proxies]]
name = "web-https"
type = "https"
localIP = "127.0.0.1"
localPort = 443
customDomains = ["www.example.com"]
[proxies.transport]
tls.enable = true

STCP — 需访问端也运行 frpc：

[[proxies]]
name = "secret-ssh"
type = "stcp"
secretKey = "my-secret-key"
localIP = "127.0.0.1"
localPort = 22

XTCP — 点对点穿透（两端直连，不经过服务端转发）：

[[proxies]]
name = "p2p-ssh"
type = "xtcp"
secretKey = "my-secret-key"
localIP = "127.0.0.1"
localPort = 22

SUDP — 安全 UDP 穿透：

[[proxies]]
name = "secret-dns"
type = "sudp"
secretKey = "my-secret-key"
localIP = "127.0.0.1"
localPort = 53

3. 传输层配置

为代理通道启用加密和压缩：

[[proxies]]
name = "ssh-secure"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 6000

[proxies.transport]
useEncryption = true
useCompression = true

全局传输配置（影响所有代理）：

transport.tls.enable = true     # 全局启用 TLS
transport.tls.serverName = "example.com"

4. 插件配置

static_file — 将目录暴露为 HTTP 文件服务：

[[proxies]]
name = "file-server"
type = "tcp"
remotePort = 7500

[proxies.plugin]
type = "static_file"
localPath = "/sdcard"
stripPrefix = "/sdcard"

socks5 — SOCKS5 代理：

[[proxies]]
name = "socks5"
type = "tcp"
remotePort = 10800

[proxies.plugin]
type = "socks5"
username = "user"       # 可选
password = "pass"       # 可选

http_proxy — HTTP 正向代理：

[[proxies]]
name = "http-proxy"
type = "tcp"
remotePort = 10801

[proxies.plugin]
type = "http_proxy"
username = "user"       # 可选
password = "pass"       # 可选

unix_domain_socket — 转发 Unix Socket：

[[proxies]]
name = "docker-sock"
type = "tcp"
remotePort = 7501

[proxies.plugin]
type = "unix_domain_socket"
unixPath = "/var/run/docker.sock"

5. Store 持久化（推荐）

FRP v0.52+ 支持 Store 模式，重启后自动恢复连接不掉线：

[[proxies]]
name = "ssh-store"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 6000

[proxies.transport]
useEncryption = true
useCompression = true

[proxies.plugin]
type = "static_file"
localPath = "/sdcard"
remotePort = 7500

Store 模式下 frpc 在本地持久化会话状态，服务端重启后自动重连。

6. 负载均衡 & 健康检查

配置多个相同服务端地址实现负载均衡：

[[proxies]]
name = "web"
type = "http"
localIP = "127.0.0.1"
localPort = 8080
customDomains = ["www.example.com"]

[proxies.loadBalancer]
group = "web-group"         # 同组内自动负载均衡
groupKey = "my-key"         # 组密钥

[proxies.healthCheck]
type = "tcp"                # tcp 或 http
intervalSeconds = 10        # 检查间隔
timeoutSeconds = 3          # 超时
maxFailed = 3               # 连续失败次数后摘除

7. 域名泛解析 & 泛域名

泛域名解析（需要服务端支持）：

[[proxies]]
name = "wildcard-http"
type = "http"
localIP = "127.0.0.1"
localPort = 8080
customDomains = ["*.example.com"]   # 匹配所有子域名

8. 宝塔 Nginx / Apache 反代

在服务端使用宝塔面板添加反代，将域名指向 FRP 服务端的 dashboard 或代理端口。

Nginx 反代配置（宝塔 → 网站 → 添加站点 → 设置 → 反向代理）：

# 宝塔面板自动生成，无需手动编辑以下代码
# 目标 URL: http://127.0.0.1:7400
# 发送域名: $host

# 如需手动编辑 nginx 配置文件：
location / {
    proxy_pass http://127.0.0.1:7400;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_buffering off;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

Apache 反代配置（宝塔 → 网站 → 设置 → 反向代理）：

# 宝塔面板操作路径相同
# 目标 URL: http://127.0.0.1:7400

# 手动编辑 httpd 配置或 .htaccess：
ProxyPreserveHost On
ProxyPass / http://127.0.0.1:7400/
ProxyPassReverse / http://127.0.0.1:7400/
RequestHeader set X-Forwarded-Proto "https"
RequestHeader set X-Forwarded-Port "443"

HTTPS 证书（宝塔自动申请 Let's Encrypt）：

  • 在宝塔「网站」→ 设置 → SSL → 申请 Let's Encrypt 免费证书
  • 勾选「强制 HTTPS」确保所有访问自动跳转 HTTPS
  • FRP 服务端 dashboard 配置了 TLS 时，反代目标改为 https://127.0.0.1:7443

反代 frpc 配置编辑器的 API（如需公网访问管理面板）：

# 仅限高级用户，建议通过 KernelSU WebView 管理
# 服务端 nginx 反代到客户端 webServer.port=7400
location /api/ {
    proxy_pass http://127.0.0.1:7400/;
    proxy_set_header Authorization "Basic $(echo -n admin:admin | base64)";
}

10. 常见问题

  • WebUI 无法打开： 从 KernelSU 模块列表点击模块的「WebUI ( < > )」按钮进入
  • 状态一直"检测中"： 关闭页面重新点击 WebUI 按钮打开
  • 配置保存失败： 检查 /data/adb/modules/CHERWIN_FRPC/conf/ 目录权限
  • 自动启动失败： 确保 service.sh 有执行权限（755）
  • 端口冲突： 远程端口 remotePort 需在服务端未被占用
  • TOML 格式错误： 注意 [[proxies]] 使用双中括号，表使用 [proxies.xxx]
